x-pack的破解（5.x）

创建时间：2018/12/28 10:54
更新时间：2018/12/28 11:31
作者：1163418857@qq.com
来源：https://zhuanlan.zhihu.com/p/38576593

明确ES的安装路径：

cd /opt/es

解压 x-pack-5.6.9.jar
  • 备份文件：

cp /opt/es/plugins/x-pack/x-pack-5.6.9.jar /usr/x-pack-5.6.9.jar.bak

  • 解压文件，先创建文件夹，因为解压之后默认不会生成根文件夹

cd /opt
mkdir x-pack

  • 复制jar包到刚才新建的x-pack目录方便解压

cp /opt/es/plugins/x-pack/x-pack-5.6.9.jar /opt/x-pack
jar -xvf x-pack-5.6.9.jar

  • 解压之后就可以删除这个jar包了
创建 LicenseVerifier.java
echo "
package org.elasticsearch.license;

public class LicenseVerifier
{
    public static boolean verifyLicense(final License license, final byte[] encryptedPublicKeyData) {
        return true;
    }

    public static boolean verifyLicense(final License license) {
        return true;
    }
}
" > LicenseVerifier.java

  • 然后将刚才新建的这个 .java 文件编译成 .class 文件

javac -cp "/opt/es/lib/elasticsearch-5.6.9.jar:/opt/es/lib/lucene-core-6.6.1.jar:/opt/es/plugins/x-pack/x-pack-5.6.9.jar" LicenseVerifier.java

  • 路径和版本号根据自己的需求进行调整
接下来就是将生成的 LicenseVerifier.class文件覆盖到x-pack-5.6.9.jar解压出来的文件夹下面

mv LicenseVerifier.class ./x-pack/org/elasticsearch/license/

将x-pack这个文件夹重新打包

jar -cvf x-pack-5.6.9.jar ./x-pack/*

将新的 jar 覆盖掉elasticsearch的插件里面的jar

mv x-pack-5.6.9.jar /opt/es/plugins/x-pack

重启elasticsearch
  • 注意：如果重启ES报错，说找不到xpackPlugin,说明打包的不合理，我试过好多次都是报错找不到class，估计是maven打包才可以。
  • 下面是我破解之后的 jar，不是 zip ,需要将该 jar 覆盖插件里面的jar包，只是5.6.9版本适用（将压缩包和.class 文件拷贝出来直接替换的）

  • 注：6.x版本在更新license之前需要关闭xpack的安全：
  • 在elasticsearch.yml配置文件中设置：

xpack.security.enabled: false

  • 配置完成之后才可以更新license，生效之后再次开启以上的设置并且开启ssl

xpack.security.transport.ssl.enabled: true

Elasticsearch----6.2.2的集群部署.md

创建时间：2018/3/7 14:30
更新时间：2018/12/28 10:08
作者：1163418857
标签：es

1.从官网下载最新的安装包      es的下载链接

2.上传到linux服务器上进行解压，tar -zxvf   elasticsearch-6.2.2.tar.gz  /opt/

3.准备java的环境，也就是jdk要有。

4.es不支持root用户进行启动，所以需要添加新用户进行操作。

useradd jack
passwd jack

（实在需要用root也可以带参数执行）
A.在运行elasticsearch时加参-Des.insecure.allow.root=true：
./elasticsearch -Des.insecure.allow.root=true
B.修改elasticsearch启动文件，添加ES_JAVA_OPTS="-Des.insecure.allow.root=true" （没验证过）

5.然后就是修改配置文件 在${es_home}/config
 vi elasticsearch.yml

```
# ---------------------------------- Cluster -----------------------------------
#
# Use a descriptive name for your cluster:
#
cluster.name: my-application
#（集群名字，同一个集群必须一致）
#
# ------------------------------------ Node ------------------------------------
#
# Use a descriptive name for the node:
#
node.name: master-es  
#（主节点名字，每个节点需要不一样）
node.master: true
#（是否竞选主节点）
node.data: true
# （是否竞选数据节点）
# Add custom attributes to the node:
#
#node.attr.rack: r1
#
# ----------------------------------- Paths ------------------------------------
#
# Path to directory where to store the data (separate multiple locations by comma):
#
#path.data: /data/es-data
#（数据路径，不配置默认在${ES-home}下面，可自己创建，可多路径）
```

3.ERROR: bootstrap checks failed
system call filters failed to install; check the logs and fix your configuration or disable system call filters at your own risk
原因：
这是在因为Centos6不支持SecComp，而ES5.2.0默认bootstrap.system_call_filter为true进行检测，所以导致检测失败，失败后直接导致ES不能启动。
解决：
在elasticsearch.yml中配置bootstrap.system_call_filter为false，注意要在Memory下面:

bootstrap.system_call_filter: false
