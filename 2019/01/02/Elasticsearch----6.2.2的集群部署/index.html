Elasticsearch----6.2.2的集群部署.md

创建时间：2018/3/7 14:30
更新时间：2018/12/28 10:08
作者：1163418857
标签：es

1.从官网下载最新的安装包      es的下载链接

2.上传到linux服务器上进行解压，tar -zxvf   elasticsearch-6.2.2.tar.gz  /opt/

3.准备java的环境，也就是jdk要有。

4.es不支持root用户进行启动，所以需要添加新用户进行操作。

useradd jack
passwd jack

（实在需要用root也可以带参数执行）
A.在运行elasticsearch时加参-Des.insecure.allow.root=true：
./elasticsearch -Des.insecure.allow.root=true
B.修改elasticsearch启动文件，添加ES_JAVA_OPTS="-Des.insecure.allow.root=true" （没验证过）

5.然后就是修改配置文件 在${es_home}/config
 vi elasticsearch.yml

```
# ---------------------------------- Cluster -----------------------------------
#
# Use a descriptive name for your cluster:
#
cluster.name: my-application
#（集群名字，同一个集群必须一致）
#
# ------------------------------------ Node ------------------------------------
#
# Use a descriptive name for the node:
#
node.name: master-es  
#（主节点名字，每个节点需要不一样）
node.master: true
#（是否竞选主节点）
node.data: true
# （是否竞选数据节点）
# Add custom attributes to the node:
#
#node.attr.rack: r1
#
# ----------------------------------- Paths ------------------------------------
#
# Path to directory where to store the data (separate multiple locations by comma):
#
#path.data: /data/es-data
#（数据路径，不配置默认在${ES-home}下面，可自己创建，可多路径）
#path.logs: /data/logs
#（日志目录）
#path.plugins: /data/plugins
#（插件目录）
#path.scripts: /data/scripts
#（脚本目录）
#
# Path to log files:
#
#path.logs: /path/to/logs
# ----------------------------------- Memory -----------------------------------
#
# Lock the memory on startup:
#
bootstrap.memory_lock: true   （ true就是不交换，核心原因：内存交换 到磁盘对服务器性能来说是 致命 的。  ）
bootstrap.system_call_filter: false
#（配合 bootstrap.memory_lock: true一起）
#
# Make sure that the heap size is set to about half the memory available
# on the system and that the owner of the process is allowed to use this
# limit.
#
# Elasticsearch performs poorly when the system is swapping the memory.
#
# ---------------------------------- Network -----------------------------------
#
# Set the bind address to a specific IP (IPv4 or IPv6):
#
network.host: 192.168.10.14
#(网络访问地址，也就是你本机要访问的地址)
#
# Set a custom port for HTTP:
#
http.port: 9200   
#（外部访问的端口号，默认9200）
#
# For more information, consult the network module documentation.
#
# --------------------------------- Discovery ----------------------------------
#
# Pass an initial list of hosts to perform discovery when new node is started:
# The default list of hosts is ["127.0.0.1", "[::1]"]
#
discovery.zen.ping.unicast.hosts: ["192.168.10.14", "192.168.10.15","192.168.10.16"]
#（集群的几个节点机器的IP地址，内网IP  可以加上端口号  IP:PORT，默认端口9300）
#discovery.zen.ping.unicast.hosts: ["172.20.10.121", "172.20.10.122","172.20.10.123"]
#
#
# Prevent the "split brain" by configuring the majority of nodes (total number of master-eligible nodes / 2 + 1):
#
discovery.zen.minimum_master_nodes: 2
#（这个是单播和多播的配置，5.x以后不支持多播，所以只能在自己机器上面的节点之间选择master，N表示参与的master数量，
#   只有一个，这个值配置为1）
#
# For more information, consult the zen discovery module documentation.
#
# ---------------------------------- Gateway -----------------------------------
#
# Block initial recovery after a full cluster restart until N nodes are started:
#
#gateway.recover_after_nodes: 3
#gateway.recover_after_time: 5m
#gateway.expected_nodes: 1
#
# For more information, consult the gateway module documentation.
#
# ---------------------------------- Various -----------------------------------
#
# Require explicit names when deleting indices:
#
#script.engine.groovy.inline.search: on
#script.engine.groovy.inline.aggs: on
#indices.recovery.max_bytes_per_sec: 100m
#indices.recovery.concurrent_streams: 10
#action.destructive_requires_name: true
```


6.配置文件配好了之后，需要对新增加的用户进行授权。
首先切换到root用户，然后进行授权，
chown -R jack:jack /opt/elasticsearch

把es目录下面每个文件夹的权限都改为你自己这个用户的权限。（包括log以及data路径文件夹）


后面基本就不会出现这类访问被限制的问题。

7.进入bin目录进行启动

./opt/elasticsearch/bin/elasticsearch
./elasticsearch -d 后台启动 
然后访问地址，172.20.10.10:9200 然后就能看到


这就说明你成功了。


（PS：需要注意的几个点：
目录的权限，还有path.data如果没有配置，去默认的es主目录下的data里面，删除掉里面的全部东西。

1.然后如果是报内存不足或者是虚拟内存不够等问题，需要用root用户去   /etc/security/limits.conf
比如：ERROR: bootstrap checks failed max file descriptors [4096] for elasticsearch process is too low, increase to at least [65536]
解决方案：
切换到root用户，编辑limits.conf配置文件， 添加类似如下内容： sudo vim /etc/security/limits.conf 添加如下内容: * soft nofile 65536 * hard nofile 131072 备注：* 代表Linux所有用户名称

* hard nofile 65536
* soft nofile 65536
* soft memlock unlimited
* hard memlock unlimited
#elsearch hard nproc 2048
#elsearch soft nproc 2048

保存、退出、重新登录才可生效

1.ERROR: bootstrap checks failed
max virtual memory areas vm.max_map_count [65530] is too low, increase to at least [262144]

还有一个文件  vi /etc/sysctl.conf 设置
vm.max_map_count=262144    如果还不够，就设置大点！）
临时设置：sudo sysctl -w vm.max_map_count=262144
永久修改：
修改/etc/sysctl.conf 文件，添加 “vm.max_map_count”设置
并执行：sysctl -p

常见错误：
2.ERROR: bootstrap checks failed memory locking requested for elasticsearch process but memory is not locked

解决方案： 
切换到root用户，编辑limits.conf配置文件， 添加类似如下内容： 
sudo vim /etc/security/limits.conf
添加如下内容: 
* soft memlock unlimited 
* hard memlock unlimited 
备注：* 代表Linux所有用户名称
保存、退出、重新登录才可生效 

(实际配置在  /etc/security/limits.conf)
* soft nofile 65536
* hard nofile 131072
* soft memlock unlimited
* hard memlock unlimited


3.ERROR: bootstrap checks failed
system call filters failed to install; check the logs and fix your configuration or disable system call filters at your own risk
原因：
这是在因为Centos6不支持SecComp，而ES5.2.0默认bootstrap.system_call_filter为true进行检测，所以导致检测失败，失败后直接导致ES不能启动。
解决：
在elasticsearch.yml中配置bootstrap.system_call_filter为false，注意要在Memory下面:

bootstrap.system_call_filter: false
